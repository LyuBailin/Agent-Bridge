# Agent Bridge 测试任务：个人笔记管理系统

## 任务目标

构建一个个人笔记 Web 应用，包含用户认证、笔记 CRUD、数据统计功能。

## 技术栈

- **后端**: Node.js + Express + 原生 JSON 文件存储
- **前端**: 原生 JavaScript + HTML + CSS（无框架）
- **端口**: 后端 3000，前端直接用浏览器打开 HTML 文件

## 项目结构

```
project/
├── backend/
│   ├── package.json
│   ├── server.js           # Express 服务器入口
│   ├── routes/
│   │   ├── auth.js        # 认证路由
│   │   └── notes.js       # 笔记 CRUD 路由
│   └── data/
│       └── store.json      # JSON 数据存储
├── frontend/
│   ├── index.html         # 主页面
│   ├── login.html         # 登录页面
│   ├── register.html      # 注册页面
│   ├── css/
│   │   └── style.css      # 样式文件
│   └── js/
│       ├── api.js         # API 调用封装
│       ├── app.js         # 主应用逻辑
│       ├── login.js       # 登录逻辑
│       └── register.js    # 注册逻辑
└── README.md
```

## 功能需求

### 1. 用户认证

- **注册**: 用户名、密码（bcrypt 加密存储）
- **登录**: 返回 JWT token，存储在 localStorage
- **登出**: 清除 localStorage 中的 token
- **路由保护**: 未登录访问主页面自动跳转到登录页

### 2. 笔记管理

- **创建笔记**: 标题 + 内容，关联当前用户
- **查看笔记列表**: 显示当前用户所有笔记（显示标题 + 创建时间）
- **查看笔记详情**: 点击笔记标题查看完整内容
- **编辑笔记**: 修改标题和内容
- **删除笔记**: 确认后删除

### 3. 数据统计

- **首页显示**: 当前用户笔记总数、今日新增、最近 7 天每日新增趋势（文本列表）
- **无图表库依赖**: 用纯文本展示统计

## 关键验收标准

### 功能验收

- [ ] 用户可以注册并获得 "注册成功" 提示
- [ ] 用户可以登录并跳转到主页
- [ ] 登录用户可以看到自己的笔记列表
- [ ] 登录用户可以创建、查看、编辑、删除笔记
- [ ] 未登录用户访问 index.html 自动跳转到 login.html

- [ ] 密码使用 bcrypt 加密（不得明文存储）

- [ ] 前端所有 API 调用使用统一的 api.js
- [ ] 删除操作需要前端确认
- [ ] 错误情况有友好的错误提示

### 代码质量验收

- [ ] 无悬空 import（引入但未使用）
- [ ] 无硬编码的 API URL（使用统一配置）
- [ ] 无 SQL/XSS 注入漏洞
- [ ] package.json 包含所有必要依赖

## 预设 Bug（用于测试 Agent Bridge 能力）

以下 bug 是任务设计时故意埋入的，Agent Bridge 应该能在生成后通过 semantic verify 发现：

1. **Bug-1**: api.js 中使用 `fetch` 时未处理 401 响应，可能导致死循环
2. **Bug-2**: JWT 验证中间件未检查 token 过期
3. **Bug-3**: 注册时未验证用户名是否已存在，导致覆盖
4. **Bug-4**: 前端删除笔记时未发送正确的请求方法（用 GET 代替 DELETE）

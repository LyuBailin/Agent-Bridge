## 根因分析

### BUG-001: 正则表达式问题

当前正则：
```javascript
const searchMatch = cleanedBlockText.match(/SEARCH:\s*<<<\s*([\s\S]*?)\s*>>>\s*/m);
const replaceMatch = cleanedBlockText.match(/REPLACE:\s*<<<\s*([\s\S]*?)\s*>>>\s*/m);
```

问题：
1. `([\s\S]*?)` 是非贪婪匹配，会在第一个 `>>>` 处停止
2. 如果内容中包含 `>>>`（如代码示例），会错误地在内容中的 `>>>` 处停止
3. `\s*>>>` 要求 `>>>` 前面有空白，但 `>>>` 后面必须是 `\s*` 能匹配的字符

### BUG-002: 模型生成错误

模型生成的 sr 块有时缺少 `>>>` 结束标记，例如：
```sr
REPLACE:
<<<
# 内容
---
（没有 >>> 结束标记）
```

这导致正则匹配失败，抛出 "sr block missing REPLACE" 错误。

## Bug 分析

### BUG-001: 正则表达式无法处理内容中包含 >>>

| 字段 | 内容 |
|------|------|
| 文件 | src/core/adapter/parser.js:32-34 |
| 根因 | 非贪婪匹配 `([\s\S]*?)` 在第一个 `>>>` 处停止，即使它是内容的一部分 |
| 修复 | 改用 `\n>>>` 要求结束标记独占一行 |
| 教训 | 正则表达式设计时应考虑内容的特殊性，避免歧义 |

### BUG-002: 模型生成缺少 >>> 结束标记

| 字段 | 内容 |
|------|------|
| 文件 | 模型输出 |

| 教训 | prompt 应明确说明格式要求，特别是分隔符使用 |
